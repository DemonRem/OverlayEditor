DSF overlay scenery editor

Overview

This application edits X-Plane® DSF overlay scenery packages for X-Plane 8.30 or later. It runs on Windows 2000 or later, MacOS 10.3.9 or later and Linux.

This document is a reference - it is assumed that you're familiar with X-Plane's scenery system and that you know how to create X-Plane scenery objects. If not, then you can get started by following this tutorial.

Setting up

Users of MacOS 10.3 should first install wxPython 2.5.3 or later (you don't need to install MacPython - it's already installed on 10.3).

The first time that you run the application you are asked to identify X-Plane's top-level folder (ie the folder which contains the X-Plane, Plane-Maker etc applications).

The application window

The application window consists of four parts:

Main window

This shows the objects in the loaded scenery package. By default terrain is displayed as "flattened" - you can change this with . Runways, taxiways and navaids are displayed in simplified form and cannot be edited. "Autogen" scenery objects are not displayed at all.

Navigate with the cursor keys or mouse. Hold down Ctrl or Cmd for bigger steps, or Shift for tiny steps:

Up, Down, Left, Right Move
Mouse click on window edge Move
Home, End Rotate
PageUp, PageDown Change elevation
Mouse wheel, -, = Zoom

Selected objects are highlighted in pink in the main window. To select objects:

Mouse click Select object
Mouse drag Select objects
Ctrl or Cmd and click or drag Toggle selected objects
Space Select all objects in the main window (even if they're too small to see).
Ctrl or Cmd and Space Toggle selection of all objects in the main window.

If you select a single polygon (facade, forest or exclusion region) you can click on an individual node (ie corner) to select it. The selected node is highlighted in white.

These keys affect the selected objects or the selected node. Hold down Ctrl or Cmd for bigger steps or Shift for tiny steps:

Mouse click and drag Move object or node
W, S, A, D Move selected objects or node
Q, E Rotate selected objects
R, F Increase/decrease facade height or forest density
C Centre on selected objects or node
Add new node object or node
Delete or Delete selected objects or node

You can't select or otherwise manipulate runways, taxiways or navaids (ILS, VOR, DME etc).

Status bar

This shows your current location, heading, elevation and, if you have selected an object or node, the object or node's location, heading and elevation (or average location if you have selected more than one object).

List of objects

All the objects, facades etc in the scenery package folder are listed under the heading "Objects", including those that are present in the scenery package folder but are not used in the scenery. The type of object is indicated by a symbol:

  • Object (.obj)
  • Polygon - facade, forest etc
  • Exclusion region

Objects that have been made available by scenery libraries, including X-Plane's built-in libraries, are listed under further headings in the drop-down list. Note that later versions of X-Plane come with more extensive libraries, so you you should always design using the earliest version of X-Plane that you want your scenery package to be compatible with.

You can only add objects that appear in these lists to your scenery (think of this as your "palette"):

  • Use to copy objects from another scenery package.
  • Use if you have otherwise added or modified objects in the scenery package folder, or if you have installed a new scenery library.

Differences between this application and X-Plane

This application displays terrain and objects in orthogonal projection and in a simplified form. ie:

  • No lights, lines, shading or animation. Objects that only contain lights or lines are displayed as small "placeholder" cubes so that they can be edited.
  • Forests are just displayed as an outline polygon. X-Plane allows forests to contain holes (glades), however this application does not. If you edit a forest any hole will be deleted.
  • Other polygons (lines, strings etc) are just displayed as an outline and cannot be edited.
  • Terrain textures are simplified - no beaches or overlay textures.
  • No flattening of terrain around airports. Terrain is displayed as if the X-Plane "allow sloped runways" rendering option is turned on.

X-Plane looks for scenery packages in alphabetical order in the Custom Scenery folder, and stops looking once it finds a "full" DSF for the required area. So an overlay scenery package will not be displayed if it has a name later in the alphabet than a "full" scenery package for the same area. This application lets you edit the overlay scenery package and does not warn you about this problem.

When you open or reload a scenery package this application warns about any objects that it can't read or that are missing, but lets you edit and save the scenery package anyway. The unreadable or missing objects are displayed as small "placeholder" cubes. X-Plane will crash unless you either provide implementations of unreadable and missing objects, or delete them from the scenery.

X-Plane supports having multiple versions of a single library object, and chooses which object to display at random. However, this application always displays the first version of an object that it comes across. Many objects in the global8 library exist in multiple versions - eg the library object /lib/global8/us/in_sq_90_30r.obj exists in eight different versions in X-Plane 8.40; if you use that object in your scenery package then this application will always display the same version, but X-Plane will display one of the eight versions chosen at random.

Limitations

  • Does not display autogen scenery.
  • Does not display lights or lines in objects.
  • Does not convert or edit .env files.
  • Does not edit runways, taxiways, navaids etc.
  • There is a limit to the number of objects that you can select at once. The number depends on your graphics hardware.
  • Palletised or interlaced PNG files appear plain grey (X-Plane can't read the former either).
