DSF overlay scenery editor

Overview

This application edits X-Plane® DSF overlay scenery packages for X-Plane 8.30 or later. It runs on Windows 2000 or later and MacOS 10.3.9 or later.

Setting up

Users of MacOS 10.3 should first install wxPython 2.5.3 or later (you don't need to install MacPython - it's already installed on 10.3).

The first time that you run the application you are asked to identify X-Plane's top-level folder (ie the folder which contains the X-Plane, Plane-Maker etc applications).

The application window

The application window consists of four parts:

Main window

This shows the objects in the loaded scenery package. Runways and taxiways are displayed in simplified form. Terrain and "autogen" scenery objects are not displayed.

Navigate with the cursor keys or mouse. Hold down Ctrl or Cmd for bigger steps, or Shift for tiny steps:

Up, Down, Left, Right Move
Mouse click on window edge Move
Home, End Rotate
PageUp, PageDown Change elevation
=, -, Mouse wheel Zoom

Selected objects are highlighted in pink in the main window. To select objects:

Mouse click Select object
Mouse drag Select objects
Ctrl or Cmd and click or drag Toggle selected objects
Space Select all objects shown in the main window (even if they're too small to see).
Ctrl or Cmd and Space Toggle selection of all objects shown in the main window.

These keys affect the selected objects. Hold down Ctrl or Cmd for bigger steps or Shift for tiny steps:

W, S, A, D Move selected objects
Q, E Rotate selected objects
C Centre on selected objects
Delete Delete selected objects

Status bar

This shows your current location and heading and, if you have selected an object, the object's location and heading (or average location if you have selected more than one object).

Tool bar

Create a new scenery package. This just creates a new folder in X-Plane's Custom Scenery folder.
Open an existing scenery package. All the packages in X-Plane's Custom Scenery folder are listed but you can only actually edit those that use DSF overlays (rather than ENVs or "full" DSFs).
Save the scenery package. Any existing .dsf files are backed up with the extension .dsf.bak
Add the object that is highlighted in the list of objects to the scenery. The object is added at the current location and with the current heading.
Delete the selected objects.
Undo the last edit.
Set a background image. You can use the W, S, A, D, Q, E and C keys to adjust the background image.
Reload the object, texture and apt.dat files in the scenery package. Use this if you have modified any of these files outside of this application. Also use this if you have added new objects and you want these new objects to appear in the list of objects. (Note that the DSF files are not reloaded - just re-open the package if you want to revert to the last saved version).
Import objects and their textures from another scenery package and add them to the list of objects. The imported objects are placed in custom objects or objects if either of these folders exist, otherwise in the package's top-level folder. The texture path names in the imported objects are fixed up as required.
Go to an airport.
Identify X-Plane's top-level folder. Do not change the location of X-Plane's top-level folder while you are editing a scenery package.
Show this help file.

List of objects

All the objects in the scenery package folder are listed under the heading "Objects", including those that are present in the scenery package folder but are not used in the scenery. Objects that have been made available by scenery libraries are listed under further headings.

You can only add objects that appear in these lists - use if you have added new objects to the scenery package folder or if you have installed a new scenery library.

Differences between this application and X-Plane

This application displays terrain and objects in orthogonal projection and in a simplified form - ie no lights, lines, shading or animation, and simplified terrain textures. Objects that do not contain any polygons are displayed as small "placeholder" cubes so that they can be edited.

When you open or reload a scenery package this application warns about any objects that it can't read or that are missing, but lets you edit and save the scenery package anyway. The unreadable or missing objects are displayed as small "placeholder" cubes. X-Plane will crash unless you either provide implementations of all missing objects or hand-edit the DSF file to remove all references to missing objects. X-Plane will also crash if it cannot read an object.

X-Plane supports having multiple versions of a single library object, and chooses which object to display at random. However, this application always displays the first version of an object that it comes across. Many objects in the global8 library exist in multiple versions - eg the library object /lib/global8/us/in_sq_90_30r.obj exists in eight different versions in X-Plane 8.40; if you use that object then this application will always display the same version, but X-Plane will display one of the eight versions chosen at random.

Limitations

  • Does not display autogen scenery.
  • Does not display lights or lines.
  • Does not convert or edit .env files.
  • Does not edit vectors (exclusion areas, forests etc).
  • Does not edit apt.dat files.
  • There is a limit to the number of objects that you can select at once. The number depends on your graphics hardware.
  • Cannot read palettised or interlaced PNG files (X-Plane can't read the former either).
